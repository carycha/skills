# Docker CI/CD

Integrate Docker with modern CI/CD pipelines to automate your delivery lifecycle.

## Core Principles

- **Automation-First**: Every commit should trigger a build and test cycle.
- **Security-in-Pipeline**: Vulnerability scans are part of the build, not an afterthought.
- **Efficient Caching**: Use BuildKit and platform-specific caching to minimize build times.
- **Reproducible Artifacts**: Use metadata actions to tag images with SHAs and semantic versions.

## Detailed Guidance

- **GitHub Actions**: Workflows for build/push, Buildx, and Trivy integration. See [github-actions.md](references/github-actions.md).
- **GitLab CI**: .gitlab-ci.yml configuration with DinD (Docker-in-Docker) services. See [gitlab-ci.md](references/gitlab-ci.md).
- **Best Practices & Operations**: Caching strategies, security hardening, error handling, and troubleshooting. See [best-practices-ops.md](references/best-practices-ops.md).
